# Contributing to Signal & Noise Framework

Thank you for your interest! This project is in early development and we welcome contributions.

## How to Contribute

### 1. **Try It & Give Feedback**
- Use the framework on your projects
- Report bugs or confusing parts
- Suggest improvements

### 2. **Share Your Templates**
- Create conversation node variants for different use cases
- Share path tracking patterns
- Document workflows that work for you

### 3. **Improve Documentation**
- Fix typos or unclear explanations
- Add examples
- Translate to other languages

### 4. **Build Features**
- Check [Issues](https://github.com/jptrp/signal-and-noise-framework/issues) for requested features
- Propose new features in [Discussions](https://github.com/jptrp/signal-and-noise-framework/discussions)
- Submit pull requests

## Development Setup

```bash
# Clone the repo
git clone https://github.com/jptrp/signal-and-noise-framework.git
cd signal-and-noise-framework

# Test locally
./bin/signal --version

# Test in a sample project
cd examples/sample-project
../../bin/signal status
```

## Pull Request Process

1. Fork the repository
2. Create a feature branch: `git checkout -b feature/your-feature`
3. Make your changes
4. Test with real projects
5. Commit with clear messages
6. Push and create a Pull Request

## Philosophy

Contributions should align with the framework's philosophy:

- **Simple First** - Start minimal, add complexity only when needed
- **User Control** - Don't automate without permission
- **Plain Text** - Keep it markdown and git-friendly
- **No BS** - Clear, honest tools for real developers
