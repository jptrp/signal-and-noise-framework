# Usage and Naming

## Product Name
- Signal & Noise Framework
- Use ampersand in formal contexts
- "Signal & Noise" is acceptable in short-form copy

## CLI Name
- `signal` is the CLI command and should appear in code blocks
- Do not refer to the CLI as "sn" or "s&n"

## Descriptions
Preferred one-liners:
- "Conversation-driven context and decision tracking for IDE teams."
- "Capture decisions from AI chats as durable, auditable knowledge nodes."

## Tagline Options
- "Decisions, not transcripts."
- "Make conversations durable."

## File and Directory References
- Use .signal/ when referring to the project directory
- Use "conversation node" for the individual markdown files

## Legal
- Use "Signal & Noise Framework" on first reference
- Include a link to the repository when used in public docs
