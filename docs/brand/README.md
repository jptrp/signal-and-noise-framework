# Signal & Noise Brand Framework

This is the canonical reference for how the Signal & Noise Framework is named, described, and presented.

## Purpose
- Clarify what the framework is and is not
- Provide consistent messaging across docs, web, and presentations
- Maintain a calm, precise, and credible tone

## Quick Summary
Signal & Noise is a conversation-driven context and decision tracking framework for IDE teams. It turns AI chats into durable, auditable knowledge nodes.

## What We Are
- A lightweight framework and CLI for capturing conversational decisions
- Local-first, git-friendly, and transparent
- Built for teams that want clarity without automation surprises

## What We Are Not
- A productivity app or task manager
- A knowledge base that auto-ingests content without consent
- A proprietary or cloud-locked system

## Core Messaging Pillars
1. Conversation intelligence should not disappear after the chat ends
2. Decisions are more valuable than transcripts
3. Control and transparency are non-negotiable
4. Plain text keeps teams aligned and portable
